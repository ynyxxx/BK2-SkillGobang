# 五子棋AI决策引擎

## 功能说明

这个AI引擎可以根据当前棋局状态，智能推荐最佳落子位置。支持非标准情况，包括技能导致的禁手位置。

## 核心特性

- ✅ **智能评估系统**：综合评估进攻、防守、战术价值
- ✅ **棋型识别**：识别五连、活四、冲四、活三等棋型
- ✅ **禁手支持**：支持技能造成的禁止落子位置
- ✅ **标准棋盘**：19x19标准围棋棋盘
- ✅ **可扩展架构**：易于调整评分权重和策略

## 使用方法

### 1. 安装依赖

```bash
cd backend
npm install
```

### 2. 运行测试

```bash
npm run test:ai
```

### 3. 在代码中使用

```typescript
import { GobangAI } from './gobangAI';
import { BoardState } from '../../../shared/types/game.types';

// 创建AI实例
const ai = new GobangAI(19);

// 定义棋局状态
const boardState: BoardState = {
  myPieces: [
    { x: 9, y: 9 },
    { x: 10, y: 9 }
  ],
  opponentPieces: [
    { x: 9, y: 10 }
  ],
  blockedPositions: [
    { x: 11, y: 9 }  // 技能禁手位置
  ],
  boardSize: 19
};

// 获取推荐落子
const move = ai.recommendMove(boardState);
console.log(`推荐落子: (${move.x}, ${move.y})`);
```

### 4. 自定义测试

修改 `testGobangAI.ts` 中的 `customTest` 函数：

```typescript
customTest(
  [{ x: 9, y: 9 }, { x: 10, y: 9 }],  // 我的棋子
  [{ x: 9, y: 10 }],                   // 对手的棋子
  [{ x: 11, y: 9 }],                   // 禁止位置
  19                                    // 棋盘大小
);
```

## 评分系统

AI使用以下评分权重（可在 `shared/constants/gameConstants.ts` 中调整）：

| 棋型 | 权重 | 说明 |
|------|------|------|
| 五连 | 100000 | 必胜 |
| 阻止对手五连 | 50000 | 防守关键 |
| 活四 | 10000 | 两端可延伸 |
| 阻止对手活四 | 8000 | 防守重要 |
| 冲四 | 5000 | 一端被堵 |
| 活三 | 2000 | 有威胁 |
| 阻止对手活三 | 1500 | 防守 |
| 眠三 | 500 | 潜在威胁 |
| 连二 | 100 | 布局 |
| 位置分 | 10 | 位置优势 |

## 算法策略

1. **棋型识别**：在四个方向（水平、垂直、两个斜向）识别连子情况
2. **进攻评分**：评估每个位置对己方的价值
3. **防守评分**：评估阻止对手的重要性（权重0.9）
4. **位置评分**：考虑棋盘中心和已有棋子的位置优势
5. **综合决策**：选择总分最高的位置

## 测试用例

脚本包含5个预设测试用例：

1. **开局测试**：空棋盘，返回中心位置
2. **进攻场景**：形成活四的机会
3. **防守场景**：阻止对手五连
4. **禁手场景**：考虑技能造成的禁止位置
5. **复杂局面**：攻防兼备的复杂决策

## 性能优化

- 只搜索已有棋子周围2格范围内的位置（减少搜索空间）
- 空棋盘直接返回中心位置
- 临时修改棋盘状态而非复制整个棋盘
